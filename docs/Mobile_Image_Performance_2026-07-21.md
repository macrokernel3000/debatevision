# Mobile Image Performance — 2026-07-21

本次量測使用本機靜態網站、`390 × 844` 手機 viewport。下載量以實際資產檔案大小計算，載入時間為瀏覽器快取後的本機回歸觀察值；它用來比較前端負擔，不等同每位使用者的行動網路速度。

## 原因

- 舊手機首頁把 `../assets/...` 改成 `/assets/...`。在 GitHub Pages 專案網址下會越過 `/debatevision/`，實際指向 `https://macrokernel3000.github.io/assets/...`，因此公開版活動背景永久空白。
- 首頁用 CSS background，圖片服務無法接手錯誤 fallback 與載入狀態。
- 隱藏的活動畫面仍先渲染，首頁一次下載 7 張完整活動 WebP 與 3 張 JPG 卡池封面。
- 舊活動 WebP 寬度約 1711–1927px，並非手機專用尺寸。

## 修正

- 首頁改成真正的 `<img>`，使用 480px WebP 縮圖、等比例 skeleton、淡入與兩段 fallback。
- Banner 使用 1080px WebP；首頁只預載異境求生 Banner。
- 進入活動後才載入該活動與當下可見的 256px WebP 卡池封面。
- 所有 URL 經 `image-service.js` 以頁面相對位置解析，GitHub Pages 會保留 `/debatevision/` 專案路徑。

## 前後比較

| 項目 | 修正前 | 修正後 | 差異 |
| --- | ---: | ---: | ---: |
| 首頁總資產 | 1,061,057 bytes | 552,452 bytes | -47.9% |
| 首頁圖片 | 558,965 bytes | 43,848 bytes | -92.2% |
| 首頁圖片請求 | 10 | 8 | 7 張縮圖 + 1 張預載 Banner |
| 進入異境求生所需相關圖片 | 203,657 bytes | 30,942 bytes | -84.8% |
| 點入後新增下載 | 已提前塞在首頁 | 18,328 bytes | 只下載 3 張可見卡池封面 |

異境求生的 12,614-byte Banner 已由首頁單獨預載，所以點入活動時不會再次下載。三張新增卡池封面合計 18,328 bytes。

## 瀏覽器驗證

- 修正前公開版：活動背景 URL 指向錯誤根路徑，因此沒有可完成的顯示時間。
- 修正後本機快取回歸：首頁第一張活動縮圖在 66ms 內進入 loaded 狀態；點入異境求生後 Banner 與三張卡池封面在 314ms 內就緒。
- 7 張首頁縮圖自然寬度皆為 480px；異境求生 Banner 為 1080px；卡池封面皆為 256px。
- 390px 寬度下破圖數 0、水平溢出 0。

公開 GitHub Pages 目前仍是舊版本；新檔案需經既有網站更新流程部署後，線上網址才會套用本次修正。
